# RPG Box 要件定義

## プロジェクト概要

**コンセプト**: 直感的だが自由に2D RPGが作れるブラウザサイト

- プログラミング経験がない人でも直感的に本格的なRPGを作成可能
- ブラウザベースのゲームエディター

---

## MVP定義

**目標**: 短編RPGが作れるレベル（完結したストーリー、メニュー、ショップ等を含む）

### MVP必須機能

**エディター**
| モジュール | MVP範囲 |
|-----------|---------|
| データ設定 | 全フィールドタイプ（P0/P1/P2全て） |
| マップ | ペイント、レイヤー、オブジェクト、イベント編集、プレハブ、テンプレート |
| UI設計 | キャンバス、グラフィック、シェーダー（カスタム除く）、ナビ、タイムライン |
| スクリプト | Monaco Editor、コード補完、通常/内部スクリプト、テストパネル |
| ゲーム設定 | ゲーム情報、アセット管理（フォルダ整理） |
| テストプレイ | 開始設定/変数設定パターン、デバッグ表示 |
| エクスポート | WebGL（HTML） |

**ゲームエンジン**
| 項目 | MVP範囲 |
|------|---------|
| コアシステム | マップ管理、オブジェクト管理、入力処理、ゲームループ |
| データ・スクリプト | データアクセス、スクリプト実行、セーブ/ロード |
| API（基本） | Sound（再生/停止/フェード）、Camera（追従/ズーム/パン/シェイク）、Effect（エフェクト再生/色カバー） |
| デフォルトテンプレート | 戦闘、メニュー、ショップ、セーブ/ロード、クエスト、メッセージ、タイトル、設定 |

**デフォルトアセット**
| カテゴリ | MVP |
|---------|-----|
| 既存アセット | 全て（キャラ、マップチップ、エフェクト、敵、BGM、SE等） |
| アイコン | アイテム(50-100)、スキル(30-50)、属性(8-10)、異常状態(10-15) |
| UI | カーソル、ゲージ、矢印 |
| 背景 | タイトル、戦闘(15-20種)、ゲームオーバー |

### MVP後回し

| 項目                                                                              | 理由                          |
| --------------------------------------------------------------------------------- | ----------------------------- |
| チップアニメーション                                                              | 必須ではない演出              |
| 4方向通行設定                                                                     | 基本的な通行可/不可で対応可能 |
| カスタムシェーダー                                                                | 高度な演出機能                |
| スマートガイド/グループ化                                                         | 親子関係で代用可能            |
| API拡張（環境音レイヤー、距離減衰、カメラパス、複数カメラ、ポストプロセッシング） | 基本機能で対応可能            |
| デスクトップアプリ出力                                                            | WebGLで対応可能               |
| RPG Box Cloud                                                                     | Phase 2                       |
| 内部ゲームエンジン編集                                                            | 上級者向け                    |
| マルチプレイヤーAPI                                                               | 将来対応                      |

#### データ管理

- **データ追加**: データタイプに新しいデータを追加
- **データタイプ追加**: 新しいデータタイプを作成
- **データタイプ編集**: フィールドの追加・編集

**ID体系**

```javascript
// Data.データタイプID["データID"].フィールドID
Data.character['alice'].max_hp; // キャラクター「alice」の最大HP
Data.item['potion_hp'].price; // アイテム「potion_hp」の価格
Data.skill['fire_ball'].power; // スキル「fire_ball」の威力
```

- **データタイプID**: データタイプ名（英数字+アンダースコア）
- **データID**: 各データの一意ID（ユーザー定義）
- **フィールドID**: 各フィールドの一意ID（ユーザー定義）

**データ削除時の動作**

- 参照しているフィールドを自動で null（未設定）に変更
- 影響を受けた箇所をエラー表示（どのデータのどのフィールドが未設定になったか）
- ユーザーは手動で修正

**データID変更時の動作**

- 参照しているフィールドのIDを自動で新しいIDに更新
- 影響を受けた箇所を通知（どのデータのどのフィールドが更新されたか）

#### フィールドタイプ（全実装）

| タイプ         | 説明                     | 内部保存形式   | 優先度 |
| -------------- | ------------------------ | -------------- | ------ |
| 数値           | 数値入力                 | number         | P0     |
| 文字列         | 単一行テキスト           | string         | P0     |
| テキストエリア | 改行含む文字列           | string         | P0     |
| ブーリアン     | チェックリスト形式       | boolean        | P0     |
| 選択肢         | プリセット選択肢から選択 | number (index) | P0     |
| 色             | カラーパレットから選択   | string (hex)   | P0     |
| データ選択     | 他データタイプから選択   | string (id)    | P1     |
| データリスト   | データ選択の配列         | string[]       | P1     |
| データテーブル | データとカラム値の紐づけ | {id, values}[] | P1     |
| クラス         | 定義済みクラスを項目化   | object         | P2     |
| クラスリスト   | クラスの配列             | object[]       | P2     |
| 画像           | アセットから画像選択     | string (path)  | P1     |
| 音声           | アセットから音声選択     | string (path)  | P1     |
| エフェクト     | エフェクト付き画像       | {path, config} | P2     |
| 歩行チップ     | 歩行アニメーション画像   | {path, config} | P2     |

**優先度**: 実装順序の指針（P0→P1→P2の順に実装）。MVP では全て必須。

#### フィールド共通設定

- **フィールドID**: 英数字+アンダースコア（例: "max_hp", "attack_power"）
  - スクリプトからのアクセスに使用: `Data.character["alice"].max_hp`
  - 同一データタイプ内で一意
- **フィールド名**: 表示用の名前（例: "最大HP", "攻撃力"）
- **表示条件**: シンプル等価条件（フィールド = 値）
  - 例: 「アイテムタイプ = 消耗品」の時のみ表示
  - 選択肢フィールドのみ条件対象
- **必須/任意**: チェックボックスで設定
- **デフォルト値**: フィールドタイプに応じた初期値

#### クラス（テンプレート / 変数型）

- フィールドのテンプレートとして複数データタイプで再利用可能
- 変数のカスタム型としても使用

### クラス ページ

フィールドのテンプレートおよび変数のカスタム型として使用するクラス（構造体）を定義

#### 画面構成（2カラム）

```
┌──────────────┬──────────────────────────────────────────────┐
│ クラス一覧    │  クラス編集（サイドパネル）                  │
│ [+ 新規追加]  │  [フィールド一覧 + D&D並び替え]              │
└──────────────┴──────────────────────────────────────────────┘
```

- 全フィールドタイプが使用可能
- データタイプのフィールド構成にクラスを参照として組み込み可能（変更が全体に反映）
- 変数のタイプとしても使用可能

#### データ保存

- **方式**: 自動保存（変更検知→IndexedDB保存）
- **タイミング**: デバウンス処理（例: 500ms後に保存）
- **Undo/Redo**: MVP必須（Zustand middleware使用）

#### 検索・フィルタリング（MVP）

- データ名での検索
- フィールド値でのフィルタリング

#### インポート/エクスポート（MVP）

- JSON形式でのデータ出し入れ
- プロジェクト全体 or 個別データタイプ選択可能

#### バリデーション

- **必須チェック**: 必須フィールドの空チェック
- **数値範囲**: 後回し（Phase2）

#### データ削除時の整合性

- **方式**: 警告して続行
- 「このデータは○件で使用されています。削除しますか？」
- 削除後、参照先はnullになる

#### 制限

| 項目                    | 上限 |
| ----------------------- | ---- |
| データタイプ数          | 100  |
| 1タイプあたりのデータ数 | 1000 |

#### パフォーマンス（MVP）

- **仮想スクロール**: データ一覧で大量データもスムーズ表示

#### バックアップ（Phase2）

- 自動バックアップ機能は後回し
- MVPでは手動エクスポートで対応

#### デフォルトデータタイプ

**共通: ID フィールド**

- 全データタイプに必須の ID フィールド（ユーザー定義）
- 英数字とアンダースコアのみ（例: "potion_hp_small", "hero_001"）
- 同一データタイプ内で一意
- スクリプトからのアクセスに使用

### 2. マップ（緑）

RPGエディターの要領でチップやイベントを配置

#### マップメニュー配下

| ページ               | 説明                       |
| -------------------- | -------------------------- |
| マップ編集           | マップを描く画面（メイン） |
| マップデータ         | マップ基本設定             |
| オブジェクトプレハブ | オブジェクトテンプレート   |
| イベントテンプレート | イベントテンプレート       |

#### マップ編集画面（3カラム）

```
┌───────────────┬─────────────────────────────────┬───────────────────┐
│ [マップ ▼]    │ [レイヤータブ]                   │ プロパティ        │
│ ドロップダウン │ ┌───┬─────────────────────┐   │ 選択オブジェクト  │
│ ・マップ一覧   │ │🖌│      キャンバス       │   │ の編集           │
│ ・各チップセット│ │🧹│                      │   │                   │
│ ・オブジェクト │ │🪣│                      │   │                   │
│               │ │📋│                      │   │                   │
│ （選択に応じて │ │✂│                      │   │                   │
│  内容が変わる）│ └───┴─────────────────────┘   │                   │
│               │   ↑ 透明ツールバー（左端）      │                   │
└───────────────┴─────────────────────────────────┴───────────────────┘
```

#### グリッド・レイヤー

- **グリッドサイズ**: 32x32固定
- **レイヤー**: カスタム可能（追加/削除）
- **レイヤー切り替え**: キャンバス上部のタブ
- **当たり判定**: チップごとに通行可/不可を設定

#### ペイントツール

| ツール     | 説明                         |
| ---------- | ---------------------------- |
| ペン       | 1マスずつ配置                |
| 消しゴム   | チップを削除                 |
| 塗りつぶし | 連続した同じチップを一括置換 |
| スタンプ   | 複数マスを一括配置           |
| 切り取り   | 範囲選択してコピー/移動      |

#### マップオブジェクト コンポーネント

**ビルトインコンポーネント**
| コンポーネント | 説明 |
|---------------|------|
| Transform | 位置、回転、スケール（必須） |
| Sprite | 画像表示、アニメーション（任意、なしで透明オブジェクト可） |
| Collider | 当たり判定（グリッド単位: 1x1, 1x2, 2x2等） |
| Movement | 移動パターン（固定/ランダム/ルート） |
| Variables | オブジェクト専用の変数 |
| Trigger | トリガー（複数可、種類ごとに別コンポーネント） |
| Controller | プレイヤー操作可能にする |
| Effect | エフェクト再生（完了後: 自動削除/非表示） |
| ObjectCanvas | オブジェクトUI表示（1つのキャンバス内に複数UI要素を配置、オフセット設定） |

**トリガー種類**
| トリガー | 説明 |
|---------|------|
| 常時 | 一定間隔で実行（インターバル設定） |
| トーク | 話しかけた時に実行 |
| タッチ | 接触した時に実行 |
| ステップ | 踏んだ時に実行 |
| インプット | 特定キー入力で実行 |

**カスタムコンポーネント**

- コンポーネントスクリプトで自作可能
- フィールド定義 + ライフサイクル
- オブジェクトにアタッチしてプロパティパネルで設定

**ライフサイクルメソッド**
| メソッド | タイミング |
|----------|-----------|
| onCreate | オブジェクト生成時 |
| onUpdate | 毎フレーム（Active時のみ） |
| onDestroy | オブジェクト破棄時 |
| onEnable | Active化時 |
| onDisable | Inactive化時 |

**Active/Inactive**

- `object.setActive(true/false)` で切り替え
- Inactive: 非表示 + 処理停止（存在は維持）
- show/hide とは別概念（hide は表示のみ非表示、処理は継続）

#### イベント編集

- **UI形式**: ブロック形式（アクションを積み重ねる）
- **アクション追加**: モーダル（カテゴリ分けされた一覧）

**アクションカテゴリ**:
| カテゴリ | アクション例 |
|----------|-------------|
| ロジック | 変数操作、条件分岐、ループ、中断 |
| 基礎 | 見た目変更、移動、画像/エフェクト表示、カメラ演出、音声、位置移動 |
| スクリプト | イベントスクリプト呼び出し |
| テンプレート | イベントテンプレート呼び出し |

#### マップデータ設定項目

| 項目               | 説明                                                |
| ------------------ | --------------------------------------------------- |
| ID                 | 一意のID（英数字+アンダースコア、例: "dungeon_01"） |
| マップ名           | 表示名                                              |
| マップサイズ       | 幅 x 高さ（グリッド数）                             |
| 使用チップセット   | このマップで使うチップセット                        |
| BGM                | マップのBGM                                         |
| 背景画像           | 遠景など                                            |
| カスタムフィールド | ユーザー定義の追加項目                              |

#### レイヤーとチップセットの関係

- 各タイルレイヤーは複数のチップセットを参照できる（`chipsetIds: string[]`）
- タイル ID は `"chipsetId:chipIndex"` 形式（例: `"cs_001:5"`）
- レイヤーにはエディタ表示/非表示フラグ（`visible?: boolean`）を持つ

#### チップセット

- チップセットはプロジェクト全体で管理するグローバルリソース
- **フィールド定義**: 各チップが持つプロパティのスキーマを FieldType で定義（例: 通行設定=Boolean, 足音=Select）
- **チッププロパティ**: チップごとにフィールド値を設定（スパース保存: デフォルト値のチップは省略可）
- **デフォルトフィールド**: 新規チップセット作成時に通行設定・足音を自動追加
- **通行設定**: チップごとに通行可/不可（BooleanFieldType）
- **足音設定**: チップごとに足音タイプを設定（草むら/石畳/木床/砂地など）（SelectFieldType）
- **4方向通行**: 後回し（Phase2）
- **アニメーション**: 後回し（Phase2）

#### オブジェクトプレハブ

- よく使うオブジェクトのテンプレート
- **更新時の反映**: 自動反映（個別カスタマイズ部分は除く）
- 配置後も個別にカスタマイズ可能

#### イベントテンプレート

- 引数付きの再利用可能イベント
- 例: アイテム取得（item, count, message）
- 各オブジェクトのイベントから呼び出し可能

#### マップ間移動

- **方式**: ワープオブジェクトのみ
- ステップトリガーで「マップ移動」アクション実行
- 移動先マップ + 座標を指定

#### 制限

| 項目                  | 上限      |
| --------------------- | --------- |
| マップサイズ          | 1024x1024 |
| オブジェクト数/マップ | 1000      |

#### ショートカットキー（MVP）

| キー           | 動作                 |
| -------------- | -------------------- |
| B              | ペンツール           |
| E              | 消しゴム             |
| G              | 塗りつぶし           |
| Space+ドラッグ | キャンバス移動       |
| マウスホイール | ズーム               |
| 1-9            | レイヤー切り替え     |
| Ctrl+C/V       | コピー/ペースト      |
| Delete         | 選択オブジェクト削除 |

### 3. スクリプト（黄）

JavaScriptでゲームシステムをカスタマイズ

#### スクリプトメニュー配下

| ページ                   | 説明                                     |
| ------------------------ | ---------------------------------------- |
| イベントスクリプト       | イベントから呼び出すスクリプト           |
| コンポーネントスクリプト | オブジェクトにアタッチするコンポーネント |

#### スクリプト種類

| 種類                     | 用途                   | 定義方法                   |
| ------------------------ | ---------------------- | -------------------------- |
| イベントスクリプト       | イベントから呼び出し   | UIパネルで引数設定         |
| コンポーネントスクリプト | オブジェクトにアタッチ | UIパネルでフィールド設定   |
| 内部スクリプト           | 親スクリプト専用       | 引数定義なし、直接呼び出し |

#### スクリプト階層構造

```
スクリプト一覧
├ battle_start（イベントスクリプト）
│   ├ _damage_calc（内部）
│   └ _effect_play（内部）
├ shop_system（イベントスクリプト）
│   └ _price_calc（内部）
└ HealthBar（コンポーネント）
    └ _render（内部）
```

- 内部スクリプトは親スクリプトに紐づく
- 親スクリプト内から直接呼び出し可能（import不要）

#### 画面構成（3カラム）

```
┌───────────────┬───────────────────────────────────┬─────────────────┐
│ スクリプト一覧 │       コードエディター             │ 設定パネル      │
│ （階層表示）   │                                   │                 │
│               │                                   │ [引数/フィールド]│
│ ▼ battle_start│                                   │ [+ 追加]        │
│   ├ _damage   │                                   │ 1. bg           │
│   └ _effect   │                                   │    型: [画像▼]  │
│ ▶ shop_system │                                   │    ラベル: 背景 │
│               │                    ┌─────────────┐│                 │
│ [+ イベント]   │                    │テスト条件    ││ [テストパネル]  │
│ [+ コンポーネント]│                 │（タブで開閉）││                 │
└───────────────┴───────────────────────────────────┴─────────────────┘
```

#### イベントスクリプト

- UIパネルで引数の型・ラベルを設定
- イベントブロックから呼び出し時にドロップダウン/モーダルで値指定

```javascript
// battle_start.js
export function main(bg, enemy) {
  const damage = _damage_calc(100, 50); // 内部スクリプト呼び出し
  // ...
}
```

#### コンポーネントスクリプト

- UIパネルでフィールドの型・初期値を設定
- オブジェクトにアタッチしてプロパティパネルで値編集

```javascript
// HealthBar.component.js
export default {
  maxHp: 100, // フィールド（UIで型定義）
  currentHp: 100,

  onUpdate(dt) {
    const ratio = this.currentHp / this.maxHp;
    this.object.scaleX = ratio;
  },
};
```

#### コードエディター

- **ライブラリ**: Monaco Editor（VS Code同等）
- **コード補完**: MVP（RPG Box APIの補完）
- **シンタックスハイライト**: JavaScript/TypeScript

#### 将来対応

- ビジュアルコーディング（ブロックプログラミング）
- チュートリアル機能

#### RPG Boxパッケージ（API）

**ネーミング規約**: 名前空間スタイル（RPG省略）

```javascript
Data['character'][2].name; // 短いアクセス
Data.get('character', 2); // メソッドアクセス
```

**パッケージ一覧**
| パッケージ | 説明 | 例 |
|-----------|------|-----|
| Data | データアクセス | `Data["character"][2].name` |
| Variable | 変数アクセス | `Variable.get("flag_1")` |
| Sound | 音声再生 | `Sound.playBGM("bgm_1")` |
| Map | マップ情報 | `Map.current().getObject("player")` |
| Camera | カメラ操作 | `Camera.moveTo({x:100, y:50})` |
| Tween | アニメーション | `Tween.to(obj, {x:100}, 500)` |
| UI | UI操作 | `UI.Canvas["menu"].show()` |

**UIパッケージ詳細**

```javascript
// 画面操作
UI.Canvas['menu'].show();
UI.Canvas['menu'].hide();
UI.Canvas['menu'].set(data); // カスタムファンクション
UI.Canvas['menu'].getObject('hp_bar'); // オブジェクト取得

// エレメント作成
UI.create('rectangle', { x: 0, y: 0, width: 100, height: 20 });
UI.create('text', { x: 50, y: 10, content: 'Hello' });
UI.create('image', { x: 0, y: 0, src: 'icon.png' });

// テンプレートから生成
UI.spawn('damage_number', { x: 100, y: 50, text: '128' });

// マップオブジェクトのObjectCanvas操作
const player = Map.current().getObject('player');
player.canvas.damage(128); // ファンクション実行
player.canvas.getObject('hp_bar'); // オブジェクト取得
player.canvas.spawn('damage_number', { text: '128' });
```

**Tweenプリセット**

```javascript
Tween.scaleX(obj, 1.5, 300); // X軸スケール
Tween.scaleY(obj, 1.5, 300); // Y軸スケール
Tween.scale(obj, 1.5, 300); // 全体スケール
Tween.vibe(obj, 5, 100); // 振動（揺れ）
Tween.fadeIn(obj, 300); // フェードイン
Tween.fadeOut(obj, 300); // フェードアウト
Tween.slideIn(obj, 'left', 300); // スライドイン
Tween.bounce(obj, 300); // バウンス
```

### 4. UI（ピンク）

直感的にゲームUIをデザイン

#### UIメニュー配下

| ページ             | 説明                              |
| ------------------ | --------------------------------- |
| 画面設計           | 全画面UI（メニュー、バトル等）    |
| オブジェクトUI設計 | マップオブジェクト上に表示するUI  |
| タイムライン       | アニメーションテンプレート        |
| シェーダー         | GLSL カスタムシェーダー（後回し） |

#### UI設計アーキテクチャ

**単一ランタイムエンジン方式**

- 画面設計はUI関連スクリプトのノーコード化
- 最終的にファンクション/アクション/コンポーネントは JavaScript として動作
- エディタテストとゲーム実行で **同じランタイムエンジン** を使用
- 新コンポーネント追加 = ランタイムに1回実装するだけ

```
┌─────────────────────────────────────────────────────────┐
│           共通ランタイムエンジン (JavaScript)            │
│  ・UIコンポーネント実装（Image, Text, Layout等）        │
│  ・アクション実行エンジン                               │
│  ・ファンクション呼び出し処理                           │
└─────────────────────────────────────────────────────────┘
                         ↑
              画面設計データ (JSON)
                         ↑
        ┌────────────────┴────────────────┐
   エディタテスト                      ゲーム実行
   (同じエンジン)                     (同じエンジン)
```

**エディタキャンバス構成（WebGL + DOM オーバーレイ）**

```
┌─────────────────────────────────────────┐
│  DOM レイヤー（エディタ専用）           │  ← 選択枠、ガイド線、リサイズハンドル
├─────────────────────────────────────────┤
│  WebGL キャンバス（ゲームと共通）       │  ← 実際のUI描画
└─────────────────────────────────────────┘
```

- UI描画は WebGL で統一 → エディタとゲームで見た目が完全一致
- エディタ機能（選択枠、ハンドル等）は DOM で実装

#### 画面設計（3カラム）

```
┌───────────────┬─────────────────────────────────┬───────────────────┐
│ [画面 ▼]      │           キャンバス             │ プロパティ /      │
│ ドロップダウン │    ゲーム画面範囲（赤枠）        │ ファンクション編集 │
│ ・画面一覧    │                                 │                   │
│ ・エレメント  │                                 │ （選択に応じて    │
│ ・テンプレート │                                 │  内容が切り替わる）│
│ ・ファンクション│                                 │                   │
│               │                                 │ [オブジェクト選択時]│
│ （選択に応じて │                                 │ → プロパティ表示  │
│  内容が変わる）│                                 │                   │
│               │                                 │ [ファンクション選択時]│
│               │                                 │ → ブロック編集    │
└───────────────┴─────────────────────────────────┴───────────────────┘
```

**左カラム ドロップダウン切り替え:**
| 選択 | 表示内容 |
|------|---------|
| 画面一覧 | 画面選択 + 新規作成 |
| エレメント | Image/Text/Shape等のUI要素 |
| テンプレート | 保存済みテンプレート一覧 |
| ファンクション | show/hide/set等 + 新規追加 |

**画面ファンクション**

- ブロック形式で組み立て（イベント編集と同じUI）
- show(), hide() はデフォルト提供（即時表示/非表示）
  - ユーザーが上書き可能（フェードイン等のアニメーション追加）
- カスタムファンクションを追加可能（set, damage等）
- スクリプトから `UI.Canvas["menu"].set(data)` で呼び出し

**ファンクション/アクションのテスト**
| 項目 | 仕様 |
|------|------|
| テスト起動 | ファンクション横の「▶」ボタン |
| テスト範囲 | 単一ファンクション |
| 引数入力 | アクション: コンポーネント下に表示/非表示パネル、ファンクション: 右パネル |
| 結果確認 | キャンバスにリアルタイム反映 |
| デバッグ | なし |
| リセット | テストパネルにクリアボタン |

```
┌───────────────────────────────────────────────────────────────────────┐
│ ファンクション                                                        │
│ ├─ show() [デフォルト] [▶]                                           │
│ ├─ hide() [デフォルト] [▶]                                           │
│ ├─ set(data) [ブロック編集...] [▶]                                   │
│ └─ [+ ファンクション追加]                                             │
├───────────────────────────────────────────────────────────────────────┤
│ テストパネル（右パネルで展開）                         [クリア]        │
│ 引数: data = { "hp": 80, "name": "アリス" }                          │
└───────────────────────────────────────────────────────────────────────┘
```

#### オブジェクトUI設計

マップオブジェクト上に表示するUI（HPバー、ダメージ数字、リアクション等）

```
┌───────────────┬─────────────────────────────────┬───────────────────┐
│ [オブジェクトUI▼]│       プレビュー               │ プロパティ        │
│ ・UI一覧       │    ┌─────────────────────┐     │                   │
│ ・エレメント   │    │   🧍 ← オブジェクト  │     │ 選択オブジェクト  │
│ ・テンプレート │    │   ┌───┐              │     │ の編集           │
│               │    │   │128│ ← UI重ね    │     │                   │
│               │    │   └───┘              │     │                   │
│               │    └─────────────────────┘     │                   │
├───────────────┴─────────────────────────────────┴───────────────────┤
│ ファンクション（画面設計と同様）                                      │
└─────────────────────────────────────────────────────────────────────┘
```

- 画面設計と同じ機能
- オブジェクトに重ねてプレビュー表示
- マップ編集ページからも切り替えて編集可能

#### UIオブジェクトコンポーネント

UIオブジェクトもマップオブジェクトと同様にコンポーネントを持つ（Unity方式）。

**RectTransform**（必須・1つ）

**Layoutコンポーネント**
| タイプ | 説明 |
|--------|------|

| VerticalLayout | 子を縦に並べる |
| GridLayout | 子をグリッド配置 |

**Animationコンポーネント**

- タイムラインでTweenアニメーションを定義
- 複数プロパティを同時にアニメーション可能

**Actionコンポーネント**

- クリック、ホバー等のイベントハンドラ
- ブロック形式でアクションを定義

**TemplateControllerコンポーネント**（テンプレート専用）

- テンプレートとして保存時に自動追加
- 引数（公開プロパティ）を定義
- onSpawn アクションをブロック形式で編集
- 詳細は「UIテンプレート」セクション参照

#### UIテンプレート

**テンプレート作成**

1. 親オブジェクトを右クリック →「テンプレートとして保存」
2. テンプレート名を入力
3. TemplateController コンポーネントが自動追加される

**TemplateController 編集**

- UI設計ページ左カラムに「テンプレート」タブ
- テンプレート一覧からダブルクリックで編集
- 右カラムのインスペクタで設定:
  - **引数定義**: 名前、型、デフォルト値
  - **onSpawn アクション**: ブロック形式で編集

**引数定義例**
| 引数名 | 型 | デフォルト値 |
|--------|-----|-------------|
| damage | 数値 | 0 |
| color | 文字列 | "#ffffff" |

**onSpawn アクション例**

```
[label の text を {damage} に設定]
[label の color を {color} に設定]
[0.5秒かけて Y を -20 移動（相対）]
[0.5秒かけて 透明度 を 0 に変更]
[自身を削除]
```

**使用方法**

```javascript
// 画面UIから
UI.spawn('damage_number', { damage: 128, color: '#ff0000' });

// オブジェクトキャンバスから
player.canvas.spawn('damage_number', { damage: 128 });
```

### 9. ゲームエンジン実行時

ゲームエンジンはスクリプト実行・マップ管理・データ管理の基盤。戦闘やメニュー等はスクリプト+UIで実装。

#### コアシステム

**マップ管理**

- マップデータの読み込み・描画
- レイヤー管理（複数レイヤーの重ね合わせ）
- グリッド移動（1マス32px）
- 当たり判定（チップ単位の通行可否）

**オブジェクト管理**

- オブジェクトの生成・破棄
- コンポーネントの実行（見た目、ムーブメント、各トリガー等）
- オブジェクト変数の保持

**入力処理**

- キーボード / ゲームパッド対応
- キーコンフィグ（プレイヤー設定可能）
- 入力状態の取得API（押下中、押した瞬間、離した瞬間）

**ゲームループ**

- フレーム更新（60fps想定）
- 一時停止・再開

**レイヤー構造**

```
ゲーム画面
├─ マップレイヤー（カメラ追従）
│   ├─ タイルレイヤー1（地面）
│   ├─ タイルレイヤー2（装飾）
│   ├─ オブジェクトレイヤー
│   │   └─ マップオブジェクト
│   │       ├─ Sprite
│   │       ├─ Collider
│   │       └─ ObjectCanvas（オブジェクトUI）
│   └─ タイルレイヤー3（屋根など）
│
├─ 画面UIレイヤー（カメラ固定）
│   └─ Canvas（メニュー、バトルUI等）
│
└─ カメラ
    ├─ 位置、ズーム
    └─ 追従対象
```

- マップレイヤー: カメラに追従、タイルとオブジェクトを描画
- 画面UIレイヤー: カメラ影響なし、最前面に固定表示
- ObjectCanvas: マップオブジェクトに追従するUI

#### データ・スクリプト

**データアクセス**

- データタイプへのアクセス（`Data["character"][2].name`）
- 変数の読み書き（`Variable.get("flag_1")`）
- オブジェクト変数の読み書き
- セーブデータへの永続化（スクリプトから呼び出し）

**スクリプト実行**
| タイプ | 説明 |
|--------|------|
| 通常スクリプト | イベントトリガーから呼び出し可能 |
| 内部スクリプト | スクリプト内からのみ呼び出し可能（モジュール分割用） |

- 非同期処理対応（await/async）
- スクリプト間の呼び出し

**セーブ/ロードAPI**

```javascript
// 保存
Save(slotId); // 全て保存
Save(slotId, { variables, party }); // 選択保存

// 読み込み
Load(slotId); // 即時復元（マップ遷移含む）
Load.getData(slotId); // データ取得のみ

// スロット情報
Save.getSlots(); // 全スロット一覧
Save.getMeta(slotId); // メタデータ取得
Save.setMeta(slotId, customData); // カスタムメタデータ設定
```

- 20スロット固定、縦リスト
- 保存対象: 変数、パーティ状態、現在マップ・座標、プレイ時間
- カスタムメタデータ対応

#### 音声・カメラ・エフェクト

**音声API（Sound）**

```javascript
// 即時操作
Sound.playBGM(id);
Sound.stopBGM();
Sound.playSE(id);
Sound.playVoice(id);
Sound.setVolume(type, volume); // type: "bgm" | "se" | "voice"

// アニメーション
Sound.bgm().fadeIn(duration);
Sound.bgm().fadeOut(duration);
Sound.bgm().fadeTo(volume, duration);
Sound.crossFade(fromId, toId, duration);

// 拡張
Sound.addLayer(id); // 環境音レイヤー
Sound.layer(id).fadeIn(duration);
Sound.setDistance(id, distance); // 距離減衰
```

**カメラAPI（Camera）**

```javascript
// 即時操作
Camera.setZoom(scale);
Camera.setPosition({ x, y });
Camera.follow(object);

// アニメーション（第2引数でduration指定）
Camera.zoom(scale, duration);
Camera.pan({ x, y }, duration);
Camera.shake(intensity, duration);

// 複雑なアニメーション

```

**エフェクトAPI（Effect）**

```javascript
Effect.play(id, { x, y });

```

## エディター間の連携

### データ参照（イベント内）

**変数操作でデータを取得**

- 3段階ドロップダウン: [データタイプ▼] → [データ▼] → [フィールド▼]
- 各ドロップダウンに「定数/変数」トグル
  - 定数: 名前のドロップリストから選択
  - 変数: オブジェクト変数 or 変数から動的に指定

**スクリプトからのアクセス**

```javascript
// ID指定（推奨）
Data['character']['alice'].name; // "アリス"
Data['character']['alice'].level; // 10

// インデックス指定
Data['character'][0].name; // 最初のキャラクター

// メソッド形式
Data.get('character', 'alice'); // データオブジェクト取得
Data.get('character', 'alice').name; // フィールドアクセス

// 検索
Data.find('character', { job: 'warrior' }); // 条件に合うデータ一覧
```

**ID管理**

- 各データにIDフィールドを固定で持たせる
- ユーザーが自由に設定
- 同一データタイプ内で一意（重複禁止）

### アセット共有

**管理方式**: 一元管理

- 全アセットは「ゲーム設定 > アセット」に保存
- 各エディターはアセットIDで参照
- 参照切れ時は代替画像（No Image）を表示

**フィールドタイプ別分類**

```
├── 画像参照
│   ├── 画像（一般）
│   ├── エフェクト
│   ├── 歩行チップ
│   └── タイルチップ
└── 音声参照
    ├── BGM
    ├── SE
    └── ボイス
```

**アセット選択UI（モーダルピッカー）**

- フィールド編集時にカレントディレクトリ設定可能
- モーダル内でフォルダ移動・検索可能
- 該当タイプのみ自動フィルタ表示

### リアルタイム反映

**方式**: 単一ストア + セレクター最適化 + immer

```javascript
import { create } from 'zustand'
import { immer } from 'zustand/middleware/immer'
import { persist } from 'zustand/middleware'

const useStore = create(
  persist(
    immer((set) => ({
      dataTypes: {...},
      maps: {...},
      scripts: {...},
      ui: {...},
      assets: {...},
    })),
    { name: 'rpg-box-storage' }
  )
)

// 各エディターは必要な部分のみ購読
const useDataEditor = () => useStore((state) => state.dataTypes)
const useMapEditor = () => useStore((state) => state.maps)
```

- 変更と同時に他エディターへ即時反映
- immerで自動的に新しい参照を生成（shallow比較対応）
- セレクターで必要な部分のみ購読（パフォーマンス最適化）
